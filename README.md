# Projeto Galeria Animal

## Objetivo

O projeto **Galeria Animal** tem como objetivo integrar conhecimentos de **HTML, CSS e JavaScript** por meio da criação de páginas web individuais e temáticas, reunidas em uma única página principal (`index.html`) gerenciada pelo professor.

Cada aluno será responsável por desenvolver a página de um animal de sua escolha, focando na organização de arquivos e na aplicação prática das tecnologias web.

## Estrutura do Projeto e Responsabilidades

O projeto será dividido em duas partes principais:

1. Arquivo central (`index.html`)
2. Contribuições individuais dos alunos

### 1. Responsabilidade do Professor (Criação da Estrutura Base)

O professor será responsável por iniciar o projeto e fornecer a estrutura central.

| Arquivo/Item | Descrição |
| --- | --- |
| `index.html` | Arquivo HTML principal da galeria. Deve listar o nome de cada animal e conter links para as páginas individuais dos alunos. |
| Estrutura de pastas base | Organização inicial para receber as contribuições dos alunos. |

### 2. Responsabilidade do Aluno (Contribuições Individuais)

Cada aluno deverá escolher um animal e criar uma pasta exclusiva com todos os arquivos necessários para sua apresentação.

| Estrutura da Contribuição | Descrição |
| --- | --- |
| Pasta do animal | Pasta com o nome do animal escolhido (ex: `leao`, `tartaruga`, `aguia`). |
| `animal.html` | Página principal do animal, contendo nome, habitat, características etc. |
| Pasta `css/` | Folha de estilo (`.css`) específica da página do animal. |
| Pasta `js/` | Arquivo JavaScript (`.js`) para interatividade da página. |
| Assets (imagens) | Imagens relevantes para o design da página. |

## Fluxo de Trabalho e Entrega

A organização e a colaboração são pontos-chave neste projeto.

1. **Escolha do animal:** cada aluno deve informar o animal escolhido até a data definida.
2. **Criação da estrutura:** o professor disponibiliza o `index.html` e a estrutura base no ambiente de colaboração (ex: GitHub, Google Drive compartilhado).
3. **Desenvolvimento:** cada aluno desenvolve sua página individual (`animal.html`, `css/`, `js/`) dentro da pasta do animal.
4. **Integração:** após concluir, o aluno garante que sua pasta esteja corretamente inserida na estrutura do projeto.
5. **Revisão final:** o professor faz a ligação final entre o `index.html` e as páginas dos alunos.

## Estrutura de Arquivos Final (Exemplo)

```text
index.html
leao/
	leao.html
	css/
		leao.css
	js/
		leao.js
tartaruga/
	tartaruga.html
	css/
		tartaruga.css
	js/
		tartaruga.js
```

## Requisitos Técnicos Mínimos

- **HTML:** uso de tags semânticas (ex: `<header>`, `<main>`, `<section>`).
- **CSS:** estilização responsiva básica (opcional, mas recomendada) e uso de seletores.
- **JavaScript:** inclusão de pelo menos um recurso interativo com manipulação do DOM (ex: botão para mostrar/esconder informações ou um carrossel simples).
